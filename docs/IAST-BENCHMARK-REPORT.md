## Вывод

**Python (Flask):** базовая интеграция IAST работает — агент подключается, передаёт пулы методов, уязвимости и часть компонентов регистрируются на панели. Полнота обнаружения заложенных уязвимостей на `run-6` составляет **67,7%** (Python 3.12) и **64,6%** (Python 3.13). Сильные стороны — категории без длинных цепочек taint (слабый хеш, слабый ГСЧ, cookie). Слабые — инъекции с длинными цепочками (обход пути, XSS, LDAP, десериализация); на 3.13 регрессия заметнее (cmdi/pathtraver/sqli ≈ 0%).

**Java (Spring/Tomcat):** полнота существенно выше — **63–95%** в зависимости от JDK; стабильно хорошо на JDK 17/18/20/23/24 (~94%). JDK 19/21/25 отстают (~63–70%), вероятно из‑за неполного краула или особенностей runtime. Компоненты (SBOM/SCA) регистрируются после исправления Celery-очереди `immunity-sca-task`. Маршруты API для Java пока не попадают в панель.

**Инфраструктура:** для корректной обработки SBOM на проде потребовалось добавить очередь `immunity-sca-task` в команду Celery worker; без этого 2500+ задач SCA не обрабатывались.
